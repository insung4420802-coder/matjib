# 임슐랭 가이드

감성 키워드로 검색하면, **광고를 걸러낸 진짜 후기만으로 별점을 매겨** 근처 맛집을 추천하는 가족용 도구.
네이버 검색과의 차별점 = "협찬/체험단 후기를 판별해 걸러내고, 진짜 후기의 만족도로 임슐랭 별점(1~5)을 산정".

## v11 별점 로직 (핵심)

국내 별점은 이제 **광고 판별**과 **후기 만족도**를 분리한다. 추가 AI 호출은 없다.

1. **후기 만족도 (satisfaction)** — 진짜 후기의 맛·만족·재방문과 실망·불친절·재방문 거부 신호를 분석한다. 국내 별점의 직접 근거다.
2. **진짜후기 밀도 (authenticity)** — 협찬 아닌 후기 비율. 광고밭의 추천 순위 상한을 누른다.
3. **후기 양·최신성** — 증거가 충분하고 최근인지 평가한다.
4. **키워드 적합도** — 가게명·업종과 진짜 후기만으로 검색 의도 일치 여부를 확인한다.

- 진짜 후기 0~1개: 별점 없음(`자료 부족`)
- 진짜 후기 2개: 최고 4점
- 진짜 후기 3~4개: 최고 4.5점
- 국내 1등: 진짜 후기 3개 이상인 검증 성공 장소만
- 판정 API 실패: 장소는 보여주되 별점·진짜후기·1등은 숨김

광고 판별은 2단계: (1) 휴리스틱(협찬/원고료/체험단 등 문구 + 내돈내산 신호)으로 즉시 판정,
(2) `ANTHROPIC_API_KEY`가 있으면 애매한(확률 0.3~0.7) 후기만 골라 Claude가 배치 재판정 → 정확도 향상.
키가 없거나 실패해도 (1)만으로 정상 작동.

## 구조

```
├── index.html          # 검색 UI + 카카오 장소검색 + 별점 렌더 + 즐겨찾기/히스토리
└── api/
    ├── keywords.js     # 감성어 → 실제 메뉴 키워드 변환 (Claude)
    ├── blog.js         # 네이버 블로그 원시 후기 수집
    ├── judge.js        # 임슐랭 판정 (휴리스틱 + Claude 재판정)
    └── lib/score.js    # 별점 엔진 (4축 결합)
```

## 설정 (변경 없음)
- `index.html` 상단 `KAKAO_JS_KEY`에 카카오 JavaScript 키
- Vercel 환경변수: `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`, `ANTHROPIC_API_KEY`(선택), `ANTHROPIC_MODEL`(선택)
- 선택: `APP_ACCESS_KEY`를 설정하면 최초 접속 시 공유용 접근 코드를 한 번 입력한다. 지인 외 API 무단 사용을 줄이는 용도다.
- 배포 도메인을 카카오/네이버 콘솔에 등록

## 비용
- 카카오/네이버: 무료 쿼터 범위에서 사용
- Claude: 국내 검색당 키워드 변환 1회 + 애매한 후기 배치 판정 1회(필요할 때만). 만족도 계산은 규칙 기반이라 AI 호출이 추가되지 않는다.
- 해외 현지 리뷰 번역은 여러 검색어 중 첫 검색 결과에만 실행해 Haiku 호출을 최대 1회로 제한한다.
- Google Maps Platform은 2025-03부터 월 $200 일괄 크레딧이 아니라 SKU별 월간 무료 사용량 방식이다. Google Cloud에서 예산 알림과 API 쿼터를 반드시 설정한다.

## 한계
- 카카오맵/네이버 **평점 숫자는 공식 API로 제공되지 않아** 사용하지 않는다. 임슐랭 별점은 후기 신뢰도 기반 자체 지표다.
- 광고 판별은 네이버가 주는 제목+요약 텍스트 기반이라 100%는 아니다(본문 전체는 API 미제공).
- 후기 API는 서버리스 함수 경유라, 로컬에서 파일만 열면 별점/후기가 안 뜬다. `npx vercel dev` 또는 배포 후 테스트.

## 해외 모드 (신규)

왼쪽 상단 국내/해외 토글. 해외 선택 시:
- `api/keywords.js`가 "오사카 소바 맛집"을 지역(Osaka)+메뉴(soba/そば)로 분해 (한국어/영어 입력 모두 처리)
- `api/gplaces.js`가 구글 Places 텍스트 검색으로 현지 맛집+평점+리뷰 수집, 현지어 리뷰는 Claude가 한국어 번역
- `api/blog.js`로 한국인 블로그 후기도 병행 수집
- `api/judge-overseas.js` + `score.js`의 `evaluateOverseasPlace`가 구글 평점(베이지안 보정: 리뷰 적은 고평점=관광객 함정 배제) + 리뷰 수 + 한국인 화제성 + 적합도를 종합해 별점 산정
- 길찾기는 구글맵 연결

### 구글 Places API 설정
1. [console.cloud.google.com](https://console.cloud.google.com) → 프로젝트 생성
2. **Places API (New)** 사용 설정
3. 사용자 인증 정보 → API 키 생성
4. **결제 계정 연결 필수** (신용카드). SKU별 월간 무료 호출 한도와 현재 가격은 Google 공식 가격표에서 확인
5. Vercel 환경변수 `GOOGLE_MAPS_API_KEY`에 등록

### 해외 지도 보기 (선택)
해외 모드의 [지도] 토글을 쓰려면 브라우저용 구글맵 키가 별도로 필요합니다:
1. 구글 클라우드 콘솔 → API 라이브러리 → **Maps JavaScript API** 사용 설정
2. 사용자 인증 정보 → API 키 **새로 생성** (서버 키와 분리)
3. 키 제한: 애플리케이션 제한 = **웹사이트**, 배포 도메인(`https://앱이름.vercel.app/*`) 등록 + API 제한 = Maps JavaScript API
4. `index.html` 상단 `GOOGLE_MAPS_JS_KEY`에 입력
비워두면 해외 지도 보기만 꺼지고 나머지는 정상 작동합니다. 지도는 사용자가 지도 버튼을 눌렀을 때만 로드됩니다.
