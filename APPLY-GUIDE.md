# 임슐랭 가이드 v11 적용 안내

## 1. 적용 전에 보관할 값

현재 배포 중인 `index.html`을 열어 아래 두 값을 별도 메모에 복사한다.

- `KAKAO_JS_KEY`
- `GOOGLE_MAPS_JS_KEY`

v11 파일에는 보안을 위해 실제 키가 들어 있지 않으므로, 파일 교체 후 이 값을 다시 넣어야 한다.

## 2. 가장 쉬운 GitHub 적용 방법

1. GitHub 저장소에서 **Add file → Upload files**를 누른다.
2. 이 v11 폴더 안의 파일과 폴더를 저장소 최상단에 업로드한다.
3. 같은 이름의 파일을 교체하고 새 파일을 추가한다.
4. 커밋 메시지를 `임슐랭 v11 신뢰도 개선`로 적고 커밋한다.
5. GitHub에서 새 `index.html`을 편집해 1단계에서 보관한 카카오/구글 브라우저 키를 다시 입력한다.
6. Vercel 자동 배포가 끝날 때까지 약 1~2분 기다린다.

새로 추가되는 파일:

- `api/lib/guard.js`
- `package.json`
- `tests/score.test.mjs`
- `APPLY-GUIDE.md`

## 3. Vercel 환경변수

기존 값은 그대로 둔다.

- `NAVER_CLIENT_ID`
- `NAVER_CLIENT_SECRET`
- `ANTHROPIC_API_KEY`
- `ANTHROPIC_MODEL` — 설정하지 않으면 기존처럼 `claude-haiku-4-5`
- `GOOGLE_MAPS_API_KEY`

선택 사항:

- `APP_ACCESS_KEY`: 지인에게 공유할 접근 코드

`APP_ACCESS_KEY`를 설정하면 앱이 최초 API 사용 시 코드를 한 번 묻고 기기에 저장한다. 예측하기 쉬운 4자리 숫자보다는 8자 이상의 단어+숫자 조합을 권장한다. 설정하지 않으면 기존처럼 코드 없이 동작한다.

접근 코드를 켜면 인증 우회를 막기 위해 서버 공유 캐시는 사용하지 않는다. 소수 지인에게만 공유한다면 코드를 켜는 편이 안전하고, 공개 링크이면서 반복 검색 캐시를 우선한다면 설정하지 않은 채 Google 쿼터를 낮게 잡는 편이 낫다.

환경변수를 추가하거나 바꾼 뒤에는 Vercel에서 **Redeploy**한다.

## 4. Google 비용 안전 설정

Google Maps Platform은 2025년 3월부터 월 $200 일괄 크레딧이 아니라 API 항목별 무료 호출 한도를 사용한다.

Google Cloud Console에서 다음을 설정한다.

1. **Billing → Budgets & alerts**에서 월 예산 알림을 만든다.
2. 50%, 80%, 100% 알림을 켠다.
3. **APIs & Services → Places API (New) → Quotas**에서 예상 사용량보다 약간 높은 일일 한도를 정한다.
4. 서버 키는 Places API (New)만, 브라우저 키는 Maps JavaScript API만 허용한다.

지인 공유 규모라면 무료 사용량 안에 머물 가능성이 높지만, `currentOpeningHours`와 `reviews` 같은 필드는 높은 Places 요금 등급을 사용할 수 있으므로 알림과 쿼터를 함께 설정하는 것이 안전하다.

## 5. 배포 후 확인 항목

아래 순서로 한 번씩 확인한다.

1. 국내에서 `판교 돈까스` 검색
2. 진짜 후기 0~1개인 장소가 `별점 자료 부족`으로 표시되는지 확인
3. 별점 카드에 `후기 만족도` 막대가 표시되는지 확인
4. 국내 검색 도중 해외 탭으로 바꾼 뒤 다시 검색해 앱이 잠기지 않는지 확인
5. 해외에서 `오사카 난바 라멘` 검색
6. 구글 원평점과 임슐랭 종합 별점이 구분되어 표시되는지 확인
7. 즐겨찾기에 저장한 뒤 즐겨찾기 화면에서도 별점이 남는지 확인

## 6. 문제 발생 시 되돌리기

GitHub 저장소의 **Commits**에서 v11 적용 직전 커밋을 열고 해당 파일 내용을 복원한다. 빠른 복구가 필요하면 기존 v10 폴더의 파일을 다시 업로드하고 `index.html`의 두 브라우저 키를 복원한다.

## 7. 비용 변화

- 국내 만족도 분석: 규칙 기반이므로 Claude 호출 추가 없음
- 국내 광고 판정: 기존과 동일하게 애매한 후기 최대 40건을 Haiku 1회로 묶음
- 해외 리뷰 번역: 검색어마다 호출하던 방식을 첫 검색어에만 호출하도록 축소
- AI 심사평·주문 도우미: 기존처럼 1등 또는 버튼 클릭 시에만 호출
